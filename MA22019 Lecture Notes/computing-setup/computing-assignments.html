  1. Open the .qmd file(s) provided

ImportantCommit at Least 3 Times!

Step 5: Push to GitHub

Your work is now submitted! The state of your repository at the deadline is your submission.

Workflow Points (20%)

ImportantThe 3-Commit Rule

You must commit at least 3 times as you work through each assignment. This is a graded requirement!

We grade your Git workflow to ensure you‚Äôre developing good habits:

Requirement Points What We Check
Minimum 3 commits 10% We look at your commit history
Final files in repo 5% Both .qmd and rendered output on GitHub
Organization 5% Clean repo, meaningful commit messages

Why do we require multiple commits?

  1. Proves progression: Shows you worked incrementally, not all at once
  2. Enables recovery: If something breaks, you have earlier working versions
  3. Mirrors industry: Professional developers commit frequently
  4. Protects you: We can give partial credit if your final version has issues
